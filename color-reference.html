Node Color System

Proposed reorganization: colors reflect what nodes do, not just their category

Design Principles Proposed

Principle 1: Value Sources Match Output Type
Nodes with no inputs that just provide a value should use the color of their output port type.
Pi outputs float → uses PORT_TYPES.float.color
Principle 2: Processing Nodes Reflect Their Domain
Nodes that transform data should have colors that indicate what kind of operation they perform.
Noise generation, UV distortion, color grading, etc. each get distinct visual identities.
Principle 3: Special Nodes Stand Out
Terminal nodes (Output), storage (Variables), and debug tools should be immediately recognizable.

Implementation Guide

Step 1: Add NODE_COLORS to PortTypes.js (or create NodeColors.js)
Step 2: Update each node file to import and use the appropriate color: 
// Before
"#4a3a5a",

// After
import { NODE_COLORS } from "./PortTypes.js";
// ...
NODE_COLORS.noise,
Step 3: For value source nodes (no inputs), continue using PORT_TYPES.*.color: 
// Nodes with no inputs match their output type
PORT_TYPES.float.color,  // Pi, seconds, etc.
PORT_TYPES.vec2.color,   // Front UV, etc.
